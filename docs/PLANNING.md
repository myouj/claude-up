## 一、核心定位

**工具名称**：PromptVault Personal
**产品定位**：面向个人开发者的 Prompt 实用工具
**核心原则**：
- 无登录，开箱即用
- 数据全本地存储，用户自己负责备份
- 功能做减法，核心体验做精

---

## 二、技术架构

### 前端方案：HTMX + 少量原生JS

| 方案 | 结论 |
|------|------|
| HTMX 主体交互 | ✅ 处理表单提交、版本切换、AI测试响应 |
| 原生JS (50-100行) | 负责：分栏拖拽、变量高亮、实时预览 |
| 无需 Vue/React | 降低复杂度，保持轻量 |

**为什么不用 Vue3**：CRUD 为主的应用不需要复杂的状态管理，HTMX + Go 模板渲染更直接。

### 后端方案：Go + Gin（保留 REST API）

- Go 模板直接渲染 HTML 页面（主交互）
- REST API 保留作为可选层（未来 CLI / 移动端口子）
- SQLite 本地存储，用户指定数据目录

### 数据库：SQLite

```
prompts 表
  - id, name, tags (JSON), created_at, updated_at
  - context (JSON) ← 执行上下文字段

versions 表
  - id, prompt_id, content, description (commit message)
  - context (JSON), test_result_summary
  - created_at

test_cases 表 (V1)
  - id, prompt_id, input_vars (JSON), expected_output
  - created_at

test_history 表 (V1)
  - id, version_id, input_vars (JSON), output, tokens
  - latency_ms, model, created_at
```

## 五、交互设计

### 主界面布局

```
┌─────────────────────────────────────────────────────────────┐
│  PromptVault                              [API Key] [设置]  │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────────────────────────────┐ │
│  │ 搜索框        │  │                                      │ │
│  ├──────────────┤  │                                      │ │
│  │ Prompt 列表   │  │                                      │ │
│  │              │  │                                      │ │
│  │ • Prompt A  │  │           编辑器 / 预览 / 测试         │ │
│  │ • Prompt B  │  │                                      │ │
│  │ • Prompt C  │  │                                      │ │
│  │              │  │                                      │ │
│  └──────────────┘  └──────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 变量预览交互

- **编辑侧**：输入时用浅色背景高亮 `{{variable}}`
- **右侧面板**：固定展示"渲染后"完整内容，输入变量值时实时更新
- **触发方式**：input 事件 + debounce 300ms + HTMX hx-trigger

### 测试面板

- **布局**：左右分栏，宽度可拖拽（CSS resize 或 split.js）
- **变量输入**：折叠在测试面板顶部
- **结果展示**：模型回复 + token 消耗 + 执行时间 + latency P50/P95

### 版本对比三段式

```
┌──────────────────────────────────────────────────────┐
│ 版本 v3 vs v2 对比                                    │
├──────────────────────────────────────────────────────┤
│ 【参数】 model: gpt-4o → gpt-4-turbo                │
│         temperature: 0.7 → 0.5                       │
├──────────────────────────────────────────────────────┤
│ 【Prompt】                                            │
│ - 你是一个助手 → 你是一个代码审查助手                  │
│ + 请检查以下代码的bug                                  │
├──────────────────────────────────────────────────────┤
│ 【测试输出】                                           │
│ v3: "这段代码有3处可优化..."                          │
│ v2: "代码看起来没问题"                                │
└──────────────────────────────────────────────────────┘
```

## 六、测试评估方案

### 小样本自动化回归

- 用户维护 5-10 个 test cases（输入变量 + 期望输出）
- 每次保存版本时自动跑一遍
- 对比输出 diff + 结构化评分

### 评分指标

| 指标 | 说明 |
|------|------|
| JSON Schema 校验 | 如果期望是 JSON，验证格式 |
| 关键词命中 | 检查关键token是否出现 |
| 正则匹配 | 自定义规则匹配 |
| Latency P50/P95 | 响应延迟分布 |
| Token 消耗 | 输入 + 输出 token 数 |
