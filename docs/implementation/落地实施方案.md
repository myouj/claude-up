## 一、MVP 范围定义

### 1.1 Must-have（V1 交付，6-8 周）

| 优先级 | 功能 | 负责人 | 说明 |
|--------|------|--------|------|
| P0 | 异步任务队列 | 后端 | 基础设施，所有耗时操作的基础 |
| P0 | 变量实时预览 | 前端 | 右侧独立面板，实时渲染 |
| P0 | Prompt 批量测试 | 前端 | 表格+卡片混合展示 |
| P0 | A/B 测试 + 统计显著性引擎 | 后端 | 序贯检验，最小15次，最大50次 |
| P0 | 自动评测集生成 | 后端 | AI 辅助 + 人工补充 |
| P0 | API 限流与配额管理 | 后端 | 防止批量操作失控 |
| P0 | Prompt 回归检测（轻量版） | 后端 | 保存轻量检测 + 模型升级专项 |

**核心路径**：写好 prompt → 测好 prompt → 知道哪个更好

### 1.2 Should-have（V1 后期）

| 功能 | 说明 |
|------|------|
| Prompt 质量评分卡 | 4维度：Clarity 30%、Completeness 30%、Example 25%、Role 15% |
| 多轮对话流程测试 | 对话轮次模拟，暴露上下文累积问题 |
| AI 响应缓存层 | 降低 token 消耗和响应延迟 |
| 结构化日志 + 追踪 | 团队协作可观测性 |

### 1.3 Nice-to-have（V1 完成后）

| 功能 | 说明 |
|------|------|
| 成本分析中心 | Token 预算 + 月度看板 |
| 场景标签 | 比分类更灵活 |
| 草稿自动保存 | 每10秒自动保存 |
| Chain-of-Thought 引导器 | 推理类 prompt 最佳实践 |

## 二、Sprint 里程碑规划（7 周）

```
Week 1-2 (Sprint 1): 基础设施
  → 异步任务队列 + 变量预览增强 + API 限流

Week 3-4 (Sprint 2): 核心测试
  → 批量测试 + 评测集生成 + 质量评分卡

Week 5-6 (Sprint 3): 统计分析
  → A/B 测试 + 序贯检验 + 多轮对话

Week 7 (Sprint 4): 收尾
  → 回归检测 + 缓存 + 集成测试 + Bug修复
```

### Sprint 1: 基础设施（Week 1-2）

| 任务 | 后端 | 前端 | 依赖 |
|------|------|------|------|
| 异步任务队列 | Task 模型 + Worker 池 + SSE | useTask composable | 无 |
| API 限流与配额 | Quota service | CostCenter 基础 | 无 |
| 变量预览增强 | 无（纯前端） | VariablePreviewPanel (60/40) | 无 |
| AICallLog 表 | middleware 层拦截 | 无 | 无 |
| ActivityLog 扩展 | 新增 action_type | 无 | 无 |

### Sprint 2: 核心测试（Week 3-4）

| 任务 | 后端 | 前端 | 依赖 |
|------|------|------|------|
| 批量测试 | BatchService | BatchTestTable + BatchTestCard | Task 队列 |
| 质量评分卡 | ScoringService (4维度) | QualityScoreCard (雷达图) | AI Provider |
| 自动评测集生成 | EvalService (AI辅助) | EvalSetEditor | AI Provider |
| SSE 进度推送 | Worker 每条推送 | useTask SSE 监听 | Task 队列 |

### Sprint 3: 统计分析（Week 5-6）

| 任务 | 后端 | 前端 | 依赖 |
|------|------|------|------|
| A/B 测试 Engine | SPRT 序贯检验 | ABTestSequentialPanel | Task 队列 |
| A/B 测试前端 | ABTestList + ABTestDetail | SSE 实时进度 | 后端 API |
| 多轮对话测试 | MultiRound service | MultiRoundTester | Batch test |

### Sprint 4: 收尾（Week 7）

| 任务 | 后端 | 前端 | 依赖 |
|------|------|------|------|
| Prompt 回归检测 | Regression service | RegressionAlert | EvalSet |
| AI 响应缓存 | Cache service | 无 | 无 |
| 集成测试 + E2E | 全链路测试 | Playwright E2E | 所有 Sprint |

## 三、技术架构

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        Frontend (Vue 3)                       │
│   PromptEditor │ BatchTest │ ABTest │ ScoreCard │ ...      │
│                     ↕ SSE / HTTP                             │
└────────────────────────────┬────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│                     Backend (Go/Gin)                         │
│  Handler → Service → Worker (内嵌) → Data Layer            │
│                              ↓                               │
│         ┌─────────────────────────────┐                    │
│         │  AI Provider (多模型统一)   │                    │
│         └─────────────────────────────┘                    │
└─────────────────────────────────────────────────────────────┘
```

## 九、关键风险与缓解

| 风险 | 级别 | 缓解方案 |
|------|------|----------|
| SPRT 序贯检验复杂度 | 高 | Phase 1 固定30次，Phase 2 再引入 SPRT |
| SQLite 并发瓶颈 | 中 | Worker 池控制并发；预留 PostgreSQL 升级路径 |
| AI API 成本超支 | 中 | Sprint 1 实现配额管理 + AI Call Log |
| 批量测试前端性能 | 低 | el-table 虚拟滚动（规模大时） |
| SSE 断线 | 低 | 前端自动重连（指数退避） |

## 十一、验收标准（DoD 摘要）

### 异步任务队列
- [ ] goroutine 池消费 task 表任务
- [ ] 状态：pending → running → done/failed
- [ ] SSE 实时推送进度（0-100）
- [ ] 服务重启后 running 任务恢复

### 批量测试
- [ ] CSV/JSON 上传测试用例
- [ ] 表格展示：序号|变量|模型|质量分|操作
- [ ] 点击展开完整 AI 输出
- [ ] 支持多选行 A/B 对比

### A/B 测试
- [ ] SPRT 序贯检验自动停止
- [ ] 最小15次/最大50次样本保护
- [ ] 清晰展示：置信区间、p-value、胜出

### 质量评分卡
- [ ] 4维度：Clarity 30%/Completeness 30%/Example 25%/Role 15%
- [ ] 权重可配置
- [ ] 雷达图可视化
