## 三、技术方案决策

### 3.1 异步任务队列：内嵌 Worker（单体）

**选择**：内嵌 Worker + SQLite 持久化

**理由**：
- 当前系统单体架构，引入外部队列服务增加运维复杂度
- MVP 阶段并发量可控，goroutine + channel 足够
- SQLite 存任务状态（id、type、payload、status、result、progress）
- 后续扩展只需替换实现，接口不变

**实现方案**：
```
task 表：
- id (UUID)
- type: "batch_test" | "ab_test" | "eval_gen" | "optimize"
- payload: JSON
- status: "pending" | "running" | "done" | "failed"
- result: JSON (nullable)
- progress: int (0-100)

Worker：后台 goroutine 池，poll task 表
通知：Server-Sent Events (SSE)
```

### 3.2 日志/可观测性：与现有 Activity Log 结合

**架构**：
- 扩展 activity 表，增加字段：action_type、entity_type、entity_id、detail（JSON）、trace_id
- 新增 ai_call_log 表记录 AI 调用（token、延迟、model、cost），高写入，定期归档
- 复用现有 utils/logger.go 的 trace_id 机制

### 3.3 变量校验：正则先行

**方案**：
- 匹配 `{{variable}}`：`\{\{([a-zA-Z_][a-zA-Z0-9_]*)\}\}`
- 支持 `{{variable|default}}` 默认值语法
- 提取变量名后与 Prompt 变量定义列表比对
- Phase 2 再引入 AST 处理复杂嵌套

## 四、交互方案决策

### 4.1 变量实时预览

**方案**：右侧独立面板（60% 编辑 / 40% 预览）

**理由**：
- 输入框内联预览会干扰写作节奏
- 右侧面板被动展示，用户随时瞥一眼
- 变量值输入框放在预览面板内，所见即所得

### 4.2 批量测试结果展示

**方案**：表格为主 + 卡片为辅

**理由**：
- 测试 case 超过 5 个时，表格浏览效率高于卡片
- 每行显示：序号 | 变量摘要 | 模型 | 质量分 | 操作
- 点击展开行显示完整 AI 输出（卡片形式）
- 支持多选行进行横向 A/B 对比

### 4.3 版本对比

**方案**：三段式 Diff 足够

**理由**：
- 业界标准，配合颜色标注（绿=新增，红=删除）
- 评分职责放到 A/B 评分里，不要混在一起

## 五、评测方案细节

### 5.1 A/B 测试统计显著性

**方案**：序贯检验 + 动态停止

```
1. 初始样本：每个版本至少跑 10 次
2. 序贯检验：每完成一轮，检验当前数据是否达到显著性
   - 用 Wald's Sequential Probability Ratio Test (SPRT)
   - 优点：平均所需样本量比固定样本量少 30-50%
3. 最大样本量上限：50 次（防止无限跑下去）
4. 最低样本量保护：每个版本至少 15 次

关键参数：
- 显著性水平 α = 0.05
- 统计功效 1-β = 0.80
- 最小可检测效应（MDE）：默认 10%
```

### 5.2 Prompt 质量评分卡

**维度权重**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 指令清晰度（Clarity） | 30% | 任务描述明确、无歧义 |
| 约束完整性（Completeness） | 30% | 输出格式、边界条件覆盖 |
| 示例质量（Example Quality） | 25% | Few-shot 代表性强 |
| 角色定义（Role Definition） | 15% | Agent persona 具体稳定 |

**可配置性**：用户可调整权重，不同 prompt 类型推荐不同权重模板

### 5.3 回归检测触发时机

**方案**：三个机制并行

```
保存时触发（轻量版）
- 快速对比当前版本 vs 最新历史版本
- 运行评测集前 5 条（subset）
- 30 秒内返回结果
- 低于基线 15% 以上 → 告警

定时全量扫描（深度版）
- 每天凌晨或模型版本更新时触发
- 运行完整评测集
- 低于历史平均 10% → 告警

模型升级触发（专项版）
- 检测到厂商 API 返回模型版本变化时
- 立即触发完整回归检测
- 标记受影响 prompt 列表
```
